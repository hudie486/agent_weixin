# 智能体架构与提问流程文档

## 1️⃣ 架构概览

智能体系统分为四个核心层次：


用户输入
│
▼
NLU + 意图识别层
│
├─ 普通对话意图 → 对话管理器 → 响应生成
│
└─ Skill 触发意图 → Skill 管理器 → Task 生成
│
▼
调度/执行系统
│
▼
Agent / 脚本执行
│
▼
统一反馈处理层
│
▼
输出给用户


**核心思想**：
- **NLU**：判断用户输入是普通对话还是技能调用
- **Skill 管理器**：生成标准化 Task
- **调度系统**：管理周期/触发任务
- **Agent 层**：执行任务逻辑，不直接输出结果
- **统一反馈处理层**：将 Agent 执行结果加工、格式化后反馈给用户

---

## 2️⃣ 模块说明

### 2.1 用户输入层
- 功能：
  - 收集文本/语音输入
  - 清洗与预处理
- **无需大模型参与**

### 2.2 自然语言理解（NLU）层
- 功能：
  - 意图识别（普通对话 / Skill 调用）
  - 参数抽取（时间、任务内容、目标平台）
- **大模型参与点**：
  - 对模糊或复杂表达进行理解与解析
  - 输出示例：
```json
{
  "intent": "create_reminder",
  "entities": {
    "content": "喝水",
    "interval": "5分钟"
  }
}
分角色隔离：
每个用户会话独立上下文，避免信息混淆
2.3 对话管理器
处理普通对话
维持对话状态
大模型参与：
对话生成与补全
分角色隔离：
每个会话独立上下文
2.4 Skill 管理器
管理技能模板
接收 NLU 输出，生成 Task
大模型参与：
对缺省或模糊参数进行推断
分角色隔离：
每个 Skill 执行上下文独立

Skill 示例：

class ReminderSkill:
    def create_task(self, params):
        return Task(
            type="reminder",
            content=params.get("content", "默认内容"),
            schedule=params.get("schedule", "*/5 * * * *"),
            agent="reminder_agent",
            params=params
        )
2.5 Task 对象（统一任务模型）
属性：
```
Task {
    type       # Skill 类型
    params     # Skill 参数
    schedule   # cron 或触发条件
    agent      # 执行逻辑/脚本
    metadata   # 状态、创建时间、上下文ID
}
```
与调度系统解耦
支持统一管理、查询和修改任务
无需大模型参与
2.6 调度/执行系统
功能：
管理 Task 队列
支持周期/条件触发
调用 Agent 执行 Task
大模型参与（可选）：
对复杂执行计划提供建议
分角色隔离：
每个 Task 独立上下文
2.7 Agent / 脚本执行
负责实际任务逻辑：
调用脚本、API、数据库
关键改进：
不直接输出结果
将结果传给统一反馈层
大模型参与（可选）：
动态生成脚本或执行内容
分角色隔离：
每个 Agent 执行 Task 时独立上下文
2.8 统一反馈处理层
功能：
接收 Agent 执行结果
格式化为用户可理解的自然语言
生成摘要、错误提示或可操作建议
大模型参与：
输出结果自然语言化、易读化
分角色隔离：
每个任务结果只反馈给对应会话用户
3️⃣ 流程示例

用户输入：

“每五分钟提醒我喝水”
流程：
NLU
输入 → 判定为 Skill 调用（ReminderSkill）
提取参数：content="喝水"，interval=5分钟
Skill 管理器
ReminderSkill.create_task(params) → 生成 Task
{
    "type": "reminder",
    "params": {"content": "喝水"},
    "schedule": "*/5 * * * *",
    "agent": "reminder_agent",
    "metadata": {"context_id": "user123"}
}
调度系统
注册 Task
每 5 分钟触发
Agent 执行
reminder_agent(task) 执行任务逻辑
执行结果不直接输出，而是传给统一反馈层
统一反馈处理层
将执行结果加工为自然语言
输出给用户：
“好的，我会每五分钟提醒你喝水”
4️⃣ 大模型参与总结
模块	大模型作用	参与方式	备注
NLU	意图判定、参数抽取、模糊解析	推理解析	必需
对话管理器	对话生成、补全	生成自然语言	必需/增强
Skill 管理器	参数推断	推断缺省参数	可选增强
Agent / 脚本执行	动态生成脚本或 API 调用	生成执行内容	可选增强
统一反馈处理层	结果自然语言化、摘要生成	格式化输出	必需
5️⃣ 分角色隔离策略
每个用户会话独立上下文
Skill 执行上下文独立
Task 与 Agent 绑定上下文
输出反馈严格对应会话用户
6️⃣ 可扩展性设计
新 Skill → 只需实现 create_task 与 Agent 执行逻辑
新触发类型 → 扩展调度系统
多平台消息 → Task metadata 记录 target
对话补全 → LLM 动态生成缺失参数
7️⃣ 总结

智能体流程：

用户输入 → NLU判定 → Skill触发 → Skill生成Task → 调度系统 → Agent/脚本执行 → 统一反馈 → 输出用户
大模型参与点：NLU、对话管理、参数推断、脚本生成、反馈生成
分角色隔离：用户上下文、Task上下文、Agent执行上下文
优势：自然语言调用技能、任务可扩展、周期/触发灵活、脚本结果安全处理
```
